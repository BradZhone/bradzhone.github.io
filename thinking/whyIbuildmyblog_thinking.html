# 为什么我要构建自己的博客

# 1. 记录方式的转变

​ 回顾一下自己走过的求学路，从初高中到本科，再到考研到读研到实习，知识的积淀总是离不开总结记录。读研前的我一直都是采用纸和笔记笔记的形式，要么记在课本上，要么记在专门的笔记本上，也以此为由“收藏”了许许多多不同颜色品牌功能粗细的笔和各式各样花里胡哨的辅助文具；考研备考阶段，虽然很多同学推荐使用平板电脑，但那时的我还是没舍得花这个钱，只是告诉自己，我已经习惯了纸笔记录，看纸质书不论手感观感都要好过看电子屏幕；直到读研，有了点小补贴才舍得体验体验无纸办公的感觉，从这时起，才突然感慨，“真方便啊，几百克的重量就能装入无数的书籍（ps：其实还是会受到容量限制的好吧），出门也不需要背一大堆的书和文具了，如果当初我，那么现在……”，可随着需要学习的课程越来越少以及课程越来越项目化，对平板的需求好像不再像学理论知识那么大了，现在它更多的职责是休闲娱乐、画点草图、记点手记、偶尔尝试一下板绘；到了实习期间，因为是项目驱动，使用电脑办公，使用以上记录方式的机会就更少了，虽然尝试过在公司发的笔记本上做笔记，在平板草稿本上绘制图例，但也没坚持多久。不过公司的工程师文化以及乐于分享知识的氛围还是很大程度影响了我，从来没写过技术博客的我（ps：其实也写过两篇毫无技术含量的CSDN博客）也开始写wiki总结自己所学的新知识和记录项目操作流程，也渐渐喜欢上了这种记录方式。

# 2. 从公司WIKI到个人博客

​ 既然已经有了新的知识记录方式，那为什么还要花心思做自己的博客小站呢？公司的WIKI可能更多的是面向公司内部，是所有员工的共同知识结晶，更多面向公司业务，虽然有足够的自由度，但是总会觉得不那么自在，因为它记录的是公司的兴衰更迭和人员变更，而我作为公司的一份子，既因为内心的归属感需要为公司WIKI贡献微不足道的力量，又需要一个属于自己的记录个人成长的平台，因此便有了建一个个人博客的想法，当然其实我早早就有了这种想法，只是实习期间的工作经历促进了它的实现。

# 3. 需要更多反思和成就感

​ 不知道其他人是否会有这样一种感觉，我作为工科生，从高中毕业以后就很少接触文学方面的书籍了，随着头发日渐稀疏，对自我情感与内心想法的表达能力越来越差，虽然处理事务时会带着逻辑性的思考，但是却少了自省与反思的过程，面对身边习以为常的事情也渐渐失去了探究到底的热情，很多时候接收了网络中的各种小评论便左右摇摆，一下觉得“诶，这个人说的真对”，一下又觉得“看了这个人的说法突然觉得前面那人理解的太表层了”，结果可能再来个反转。因此自认为需要更多独立思考的过程与记录的方式，我想这也是个人博客能够带给我的另一个积极作用吧。另外，写博客还能够看到自己一点点的技术积淀，能够看到成长历程与思想的变化，这就像翻开自己小时候的奖状册，拿出珍藏的童年日记一样，许久以后也能令人思绪万千吧，看看自己从前的想法，也会有种跨越时空和从前的自己对话的奇妙体验吧。

# 4. 如何构建

​ 由于没有前端经验，也想短时间内快速实现我的这个小目标，我首先调研了一些搭博客的方式，最终选择了使用vuepress来实现一个简易版本（虽然它文档有点乱，而且版本还不稳定，可能是我菜），然后部署到Github Pages上（先薅把羊毛，觉得不错再租服务器也未尝不可），再慢慢迭代升级。当然目前也只能满足基本的记录功能，更多炫酷的功能还需要后期学习实现（这不，又有东西可写了）。

# 5. 总结

​ 不论处于什么阶段，一个人的成长都离不开积淀，只是不同的时期可能有更加合适的知识想法记录方式罢了，一定不能缺失了思考的过程与经验的总结，否则只觉得浑浑噩噩度日，日复一日做着同样的事情，缺少了发现探索享受生活的热情。
